Smart Copy Checking flexible-checking patch

Copy these files into the same paths in your existing project.
This patch changes copy checking from chapter-wise records to flexible locked checking entries:
Class -> student/subject/status/details rows.

Important:
- Includes migration 0003_flexible_copy_checking.py. Run python manage.py migrate locally and Render will run migrations during build.
- Existing data is not intentionally deleted by this code update.
- Correction requests for copy checking are no longer used; users add a new locked row instead.
- Admin can export records, attention lists, and student profiles to Excel.
- Backups page now has upload/restore for the app's JSON backup files.
